5、Agent 技术演进核心脉络

单 Agent 扩术进代路径: 单 Agent 近术运代沿功能补全与能力
优化路径推进: 早期龙虾产品内核为无记忆模块的派，仅新增记
忆功能但未做分层,存在记忆混乱、任务丢失、安全性与可监测、
可调试性不足的问题.后续返代的 Hermes 实现多维度核心优化:
一是落地记忆分层机制, 将记忆划分为 Agent 感知外部交互环境
形成的世界记忆，以及 Agent 自身交互过程中产生的主观记忆;
二是支持 skill 在任务完成过程中自我和迭代沉淀，早期 skill 均为
提前预制的"预制菜"模式,存在 skill 中毒(故意诱导的错误 skil、
来源不明易引发系统混乱崩溃的问题，对此 CloudCode 推出了
针对 8 万余个 skill 的评分系统，有效解决 skill 来源不可靠的隐
患。此外 Hermes 新增反思功能，与用户交互结束后，Agent 仍
可在世界知识学畴内异步探索更优解决方案, 实现技能持续精进，
最终达到类似自然呼吸的熟练应用状态。

-多 Agent 演进的必要性: 单 Agent 以构人存在固有缺陷，仪能履
凑简单通用需求,无法文撑复杂精确任务，需向多 Agent 团队化
形态演进。 当前通用以构的单 Agent 缺乏垂直领域专属技能与知
识储备，容错性薄弱，类比单一内科医生开设的个体诊所，仪能

处理输液、售卖感冒药等简单通用需求，无法完成复杂手术这类
对专业性、精确度要求极高的任务,这是单 Agent 以构层面的固
有狂锋。因此技术演进方向指向多 Agent 协作生态，类比从个体
诊所升级为专科医院、三甲医院的逻辑，由不同角色、定位的
Agent 组成协作团队，各自承担对应领域专业任务，共同完成复
来高精确度的场景需求，从根本上弥补单 Agent 的能力短槐。

6、大三布局与俊本押注方向

科近大三核心布局: 科技大三在多智能体协作网络领域已开展多
项探索实践，微软于会议名开 3 天前推出 Agent 365，明确云昌
面环境下 Agent 的启动、运行、结束规则，云桌面 Agent 队列
与本地桌面队列互为镜像。LangChain 推出的 LangGraph 是多
智能体编排器的和雏形，具备类规则大脑功能，可完整记录 Agent
间任务授权、问题解决、结果返还的全流程，类似人类工作授权
逻辑, 可为多智能体系统提供清晰的权限、日志与历史记录文撑，
有效提升系统可审计性。

行业投资轮次与机会: 当前多智能体行业投资处于硬件层优先受
益阶段，“卖铲子”的硬件厂商是当前核心受葵方，代表厂商如
英伟达; 黄仁勋是本轮 Al 滔潮的最大获答者，其手持充沛现金反
向持股多家大模型厂商，还前瞻性收购玻璃六，底层逻辑是多智
能体系统运行需海量互联互通数据传输，原有铜线产能与传输能
力无法广撑巨量需求，玻璃可用于生产光纤，是核心通信基础设
施原材料。当前行业正处于平台层机会窗口期，已有多方布局

token 调用入口相天中转站项目，平台层核心机会集中在 token
调用入口、数据要素等赛道，待核心生产要素完全明确后，应用
层机会将逐步显现。

头部机构押注方向: 头部创投机构在多智能体领域的押注方向各
有侧重: a. 红杉聚焦 500 强巨头未覆盖的垂直细分赛道，重点
布局金融、医疗、供应链、客服等领域的Agent 项目, 采用 MAD
护城河框架评估项目，其中 M 指优先拿下目标市场，A 指保障
Agent 的可调试性、可审计性，避免架构和迭代时数据混乱难以适
配新架构，D 指以快速和迭代能力紧跟技术变化，做随技术滔潮上
浮的“船”而非被淹没的“柱子”; b. a16z 侧重布局基础设施
与编排平台赛道，相天项目关联硬件、数据及美股上市企业，可
直接落地投资动作，建议基金经理重点关注该方向; c. YC 采用
批量旷化模式，重点布局消费级 Agent 项目，通过大量试错科选
优质标的。

7、成熟Agent 架构核心特征

.成熟架构核心特性: 成熟 Agent 架构具备编排屋、多端协作、

开放协议、友好开发体验、动态化五大核心特征。各特征核心要
求与作用如下:

a. 清晰编排层: 承担调度、状态管理职能，核心作用是为任务失
败兜底，避免任务 出问题后直
接停滞。

b. 原生多端协作文持: 微软推出的AutoGen 在 2025 年 12 月

推出第二个版本后即停更，核心问题是其串联式架构会逐级放大
上游 Agent 的信息偏差，属于拼凑式架构; 成熟架构为广场式意
图串联模式，在编排器统一管控下, 多 Agent 可开展多轮分析沟
通与观点攻防，基于同一数据源形成共识，避免偏差传导。

c. 开放协议绑定: 当前多数 NG 架构采用内部 MCP 协议实现插
件调用, 仅支持内部生态资源调取, 未实现 Agent 生态内的开放
陌生友现与交易撮合，该能力可充分释放网络效应，是未来核心
发展方向。

d. 优秀开发者体验: 成熟工具需具备低调用门槛特性，当前
LangGraph 虽接近相关要求，但因属于 LangChain 产品系人询，
开上友体验不足，且未开源，可观测性存在短板。

e. Agent 动态化: Agent 的技能需具备动态迭代能力，例如
Hermes 的 skill 可通过后台汉队列开展强化学习优化能力; 同时
需配套企业级规则管控 Agent 的启停时机, 避免其过度调用服务
器、设备权限或常驻进程，实现 Al 进程动态调度。
行业发展节点判断: Agent 行业“从零到一”节点为技术与人类
生产生活完全融合的时刻，当前仍处于近术积累阶段。当成熟
Agent 以构的五大核心特征全部落地，即以构完全与人类生产生
活场景融合、达到"与水面齐平"的状态时，即为行业发展的"从零
到一"核心节点,标志着 AlGC 领域完成从无到有的核心突破。当
前扩术仍处于演进过程中，判定行业迈入成熟节点的核心信号尚
未出现，相关架构尚未达到全场景融合要求，行业整体处于向通

用架构演进的技术积累阶段，目前仪完成"从零到零点一"的前期
探索，下一个阶段为产品化。
